- **1.9** Corrección de silent-failure + tooling de calidad completo + nota de traducción multiposición (2026-05-07) :
  - **Nota de traducción multiposición + formato marker "embed card"** :
    - Nuevas opciones CLI (aditivas, valores predeterminados sin cambios → **non breaking**) :
      - `--note_position {top,bottom,both}` (predeterminado : `bottom`) : coloca la nota arriba, abajo, o en ambos lugares del archivo traducido.
      - `--note_format {legacy,marker}` (predeterminado : `legacy`) :
        - `legacy` reproduce estrictamente el comportamiento v1.8 (párrafo en negrita `**…**`) **byte-for-byte**.
        - `marker` emite una definición de referencia de enlace Markdown invisible (`[ai-translation-note-<placement>]: <> "v=1 source=… target=… model=… date=…"`) seguida de un **blockquote de 3 párrafos** estructurado para un renderizado tipo "GitHub repo embed card" : título del proyecto en código inline (`**\`ai-powered-markdown-translator\`\*\*`), descripción traducida por el LLM, y enlace CTA (`[Voir le projet sur GitHub ↗](URL)`) con flecha visible. Explotable en el build mediante un plugin remark (cf. blog jls42.org → plugin `remark-translation-banner`).
    - **Invariantes nunca enviados al LLM** : el título del repo y la URL de GitHub se ensamblan del lado Python después de traducir la frase descriptiva. El LLM nunca ve ni el slug `ai-powered-markdown-translator` ni `https://github.com/jls42/...`, garantizando que ningún renderer/case/scheme se altere.
    - **Inserción compatible con frontmatter** : en modo `top` o `both`, la nota se inserta **después del bloque de `---` de cierre** del frontmatter YAML (seguridad Astro Content Collections / gray-matter). Helper `_split_frontmatter` detecta `---\n…\n---\n` al inicio del archivo y preserva su integridad ; **lanza `RuntimeError`** sobre un frontmatter abierto sin fence de cierre (el archivo sube a `failed_files` en lugar de escribirse con una nota mal colocada).
    - **Sanitizador de modelo whitelist** : `_sanitize_model` reemplaza cualquier carácter fuera de `[A-Za-z0-9._:/-]` por `_`, fallback `unknown` si está vacío. Se alinea con el validador del lado del plugin remark de Astro y neutraliza los caracteres que romperían el formato del marker (espacio, comilla, paréntesis, coma, etc.).
    - **Refactor interno** : `_append_translation_note` (1 función monolítica) → 7 helpers puros (`_translation_note_invariants`, `_build_translation_note_phrase`, `_assemble_translation_note_paragraphs`, `_build_translation_note_source`, `_sanitize_model`, `_quote_lines`, `_split_frontmatter`, `_build_translation_note_block`, `_compose_with_notes`). Builder/composer separados (el builder devuelve un bloque puro sin separador, el composer aplica los `\n\n` según la posición) ; producción y helper source comparten el mismo ensamblador de 3 párrafos.
    - **`_quote_lines` blank-preserving** : prefija cada línea con `> `, transformando las líneas vacías en `>` únicamente. Permite a mdast ver 3 párrafos distintos en el blockquote (título / descripción / enlace) en lugar de un solo párrafo con saltos de línea.
    - **`_build_translation_note_block` adaptativo** : según el número de párrafos que el LLM haya preservado (3 = formato card completo, 2 = frase + enlace, 1 = fallback). El fallback de 1 párrafo **ya no envuelve en `**...**`** cuando se detecta un enlace Markdown `](` (renderizado frágil de `<strong>` alrededor de un enlace).
    - **Compatibilidad ascendente** : `getattr(args, "note_position", "bottom")` y `getattr(args, "note_format", "legacy")` del lado de `_compose_with_notes` — los Namespace sin esos atributos (tests existentes, llamadas programáticas externas) siguen funcionando sin modificación.
  - **Corrección de silent-failure en traducciones largas** :
    - Validación de idioma post-traducción en todos los providers (OpenAI, Mistral, Claude, Gemini) : capa determinista (extracto de origen recuperado verbatim) + capa probabilística (`langdetect`)
    - Whitelist `finish_reason` / `stop_reason` : lanzar `RuntimeError` sobre cualquier estado fuera de la whitelist (truncation, content_filter, etc.)
    - `max_tokens` Claude : `4096` → `32768` (evita truncation latente en segmentos 16k, margen cross-script FR→JA/ZH/KO/AR/HI)
    - Segmentación heading-aware : prioridad H2/H3 en la 2.ª mitad del segmento (cada segmento comienza por una sección semántica completa)
    - Propagación de errores hasta código de salida no cero : `translate_markdown_file` devuelve un estado tipado `success` / `failure` / `skipped`, `main()` `sys.exit(1)` si al menos un archivo ha fallado (single-file y batch)
    - Empty-content guard en todos los providers, sanity ratio source/output (≥ 500 chars, < 5% = rechazo), validación de placeholders de código (`#CODEBLOCK`/`#INLINECODE`), normalización post-LLM (separadores/enlaces pegados a un heading), `BadRequestError` retry sin `reasoning_effort`
    - Añadida dependencia `langdetect==1.0.9`
  - **Tooling de calidad pre-commit** ("tipo EurekAI completo", 14 hooks) :
    - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 API keys protegidas), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, etc.)
    - Pre-push : mypy (modo laxo progresivo), Opengrep SAST (translate.py + scripts/), pip-audit (modo reporting inicial), unittest discover (tests/ + scripts/tests/)
    - Wrappers locales en `scripts/` que usan `./venv/bin/python`
    - `scripts/audit_verdict.py` : parser JSON de pip-audit con 11 tests unittest, puerto Python adaptado del parser jls42-astro
    - 7 violaciones ruff iniciales corregidas : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
    - Lizard excluye temporalmente `translate.py` (4 funciones con CCN 21-47, refactor planificado) — gate estricto sobre scripts/
  - **SonarCloud + cobertura exhaustiva** :
    - Workflow GitHub Actions `SonarCloud` (sonarcloud.yml + sonar-project.properties) : análisis en cada push y pull-request, coverage vía `coverage.xml`
    - 11 badges SonarCloud en la parte superior del README (Quality Gate, Security/Reliability/Maintainability ratings, Coverage, Vulnerabilities, Bugs, Code Smells, Duplicated Lines, Technical Debt, Lines of Code)
    - `tests/test_silent_failure.py` (`unittest` stdlib) : cubre los seis eslabones de la cadena de error silent-failure
    - `tests/test_orchestration.py` (+79 tests) : cubre la capa de orquestación de `translate.py` (`_resolve_*_filename`, `_existing_translation_exists`, `_record_translation_status`, `_write_output_file`, `translate_directory`, `_validate_input_paths`, `_init_*_client`, `_select_provider_client`, `_normalize_collapsed_markdown`, `_cleanup_source_flag`, `_validate_news_flags_*`, `_openai_create_with_fallback` TypeError + BadRequestError fallbacks, o1-series prompt format, ramas early-return de `_validate_translation_output`)
    - `scripts/tests/test_audit_verdict.py` : cobertura de `main()` (stdin/stdout) y del bloque `if __name__ == "__main__"` vía subprocess
    - **Coverage on new code** : 75.5% → ~98% (translate.py 98%, scripts/audit_verdict.py 97%)
  - **Tests** : `tests/test_translation_note_position.py` cubre la matriz posición × formato (incl. E2E `marker+top|bottom|both` y `legacy+top|bottom|both`), el prefijado multilínea, la retrocompatibilidad byte-for-byte (golden literal), el sanitizador, el split frontmatter (incl. raise sobre fence no cerrado), el formato de 3 párrafos, el fallback de 2 párrafos, el guard de 1 párrafo + enlace Markdown, y un guardafuego crítico `TestLLMPayloadExcludesInvariants` que afirma que título+URL nunca se envían al LLM. **190 tests pass**, 0 regresión.
  - Documentación : `README.md` (FR + 14 traducciones) con badges, `CLAUDE.md` (workflow pre-commit + watch CI detallado), 28 traducciones regeneradas
