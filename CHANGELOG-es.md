- **1.9** Corrección de silent-failure + tooling de calidad completo (2026-05-03) :
  - **Corrección de silent-failure en traducciones largas** :
    - Validación de idioma post-traducción en todos los providers (OpenAI, Mistral, Claude, Gemini) : capa determinista (extracto de la fuente encontrado verbatim) + capa probabilística (`langdetect`)
    - Whitelist `finish_reason` / `stop_reason` : lanzar `RuntimeError` sobre cualquier estado fuera de la whitelist (truncation, content_filter, etc.)
    - `max_tokens` Claude : `4096` → `16384` (evita truncation latente en segmentos de 16k chars)
    - Segmentación heading-aware : prioridad H2/H3 en la 2.ª mitad del segmento (cada segmento comienza con una sección semántica completa)
    - Propagación de errores hasta un exit code no cero : `translate_markdown_file` devuelve un estado tipado `success` / `failure` / `skipped`, `main()` `sys.exit(1)` si al menos un archivo falló (single-file y batch)
    - Empty-content guard en todos los providers, sanity ratio source/output (≥ 500 chars, < 5% = rechazo), validación de placeholders de código (`#CODEBLOCK`/`#INLINECODE`), normalización post-LLM (separadores/enlaces pegados a un heading), `BadRequestError` retry sin `reasoning_effort`
    - Añadida dependencia `langdetect==1.0.9`
  - **Tooling de calidad pre-commit** ("type EurekAI completo", 14 hooks) :
    - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 API keys protegidas), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, etc.)
    - Pre-push : mypy (modo lax progresivo), Opengrep SAST (translate.py + scripts/), pip-audit (modo reporting inicial), unittest discover (tests/ + scripts/tests/)
    - Wrappers locales en `scripts/` que usan `./venv/bin/python`
    - `scripts/audit_verdict.py` : parser JSON pip-audit con 11 tests unittest, puerto Python adaptado del parser jls42-astro
    - 7 violaciones ruff iniciales corregidas : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
    - Lizard excluye temporalmente `translate.py` (4 funciones a CCN 21-47, refactor planificado) — gate estricto sobre scripts/
  - **SonarCloud + cobertura exhaustiva** :
    - Workflow GitHub Actions `SonarCloud` (sonarcloud.yml + sonar-project.properties) : análisis en cada push y pull-request, coverage vía `coverage.xml`
    - 11 badges SonarCloud en la parte superior del README (Quality Gate, Security/Reliability/Maintainability ratings, Coverage, Vulnerabilities, Bugs, Code Smells, Duplicated Lines, Technical Debt, Lines of Code)
    - `tests/test_silent_failure.py` (`unittest` stdlib) : cubre los seis eslabones de la cadena de error silent-failure
    - `tests/test_orchestration.py` (+79 tests) : cubre la capa de orquestación de `translate.py` (`_resolve_*_filename`, `_existing_translation_exists`, `_record_translation_status`, `_write_output_file`, `translate_directory`, `_validate_input_paths`, `_init_*_client`, `_select_provider_client`, `_normalize_collapsed_markdown`, `_cleanup_source_flag`, `_validate_news_flags_*`, `_openai_create_with_fallback` TypeError + BadRequestError fallbacks, formato de prompt o1-series, ramas early-return de `_validate_translation_output`)
    - `scripts/tests/test_audit_verdict.py` : cobertura de `main()` (stdin/stdout) y del bloque `if __name__ == "__main__"` vía subprocess
    - **Cobertura sobre código nuevo** : 75.5% → ~98% (translate.py 98%, scripts/audit_verdict.py 97%)
  - Documentación : `README.md` (FR + 14 traducciones) con badges, `CLAUDE.md` (workflow pre-commit + watch CI detallado), 28 traducciones regeneradas
