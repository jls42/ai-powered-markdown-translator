- **1.8.3** SonarCloud + cobertura exhaustiva (2026-05-03) :

  - 11 badges de SonarCloud en la parte superior del README (Quality Gate, Security/Reliability/Maintainability ratings, Coverage, Vulnerabilities, Bugs, Code Smells, Duplicated Lines, Technical Debt, Lines of Code)
  - Nuevo archivo `tests/test_orchestration.py` (+79 tests) cubriendo la capa de orquestación de `translate.py` : `_resolve_*_filename`, `_existing_translation_exists`, `_record_translation_status`, `_write_output_file`, `translate_directory`, `_validate_input_paths`, `_init_*_client`, `_select_provider_client`, `_normalize_collapsed_markdown`, `_cleanup_source_flag`, `_validate_news_flags_*`, `_openai_create_with_fallback` (fallbacks TypeError + BadRequestError), formato de prompt de la serie o1, ramas de retorno anticipado de `_validate_translation_output`
  - `scripts/tests/test_audit_verdict.py` ampliado : cobertura de `main()` (stdin/stdout) y del bloque `if __name__ == "__main__"` vía subprocess
  - **Cobertura en el nuevo código** : 75.5% → ~98% (translate.py 98%, scripts/audit_verdict.py 97%)
  - Endurecimiento adicional de `translate.py` (feedback de revisión de PR) : guardia de contenido vacío en todos los proveedores, ratio de sanidad fuente/salida (≥ 500 caracteres, < 5% = rechazo), validación de placeholders de código (`#CODEBLOCK`/`#INLINECODE`), normalización post-LLM (separadores/enlaces pegados a un heading), retry de `BadRequestError` sin `reasoning_effort`
- **1.8.2** Herramientas de calidad pre-commit (2026-04-30) :
  - Configuración `pre-commit` "tipo EurekAI completa" : 14 hooks distribuidos en dos etapas (pre-commit rápido + pre-push pesado)
  - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 claves API protegidas), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, etc.)
  - Pre-push : mypy (modo laxo progresivo), Opengrep SAST (translate.py + scripts/), pip-audit (modo de informe inicial), unittest discover (tests/ + scripts/tests/)
  - Wrappers locales en `scripts/` que usan `./venv/bin/python` (el sistema no tiene `python` en bruto fuera del venv)
  - `scripts/audit_verdict.py` : parser JSON de pip-audit con 11 tests unittest, puerto Python adaptado del parser jls42-astro
  - 7 violaciones iniciales de ruff corregidas : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
  - Documentación : README.md (FR) + CLAUDE.md (workflow detallado), 28 traducciones regeneradas
  - Lizard excluye temporalmente `translate.py` (4 funciones con CCN 21-47, refactor planificado en una PR dedicada) — gate estricto sobre scripts/ para evitar regresiones
- **1.8.1** Corrección de silent-failure en traducciones largas (2026-04-28) :
  - Validación de idioma post-traducción en todos los proveedores (OpenAI, Mistral, Claude, Gemini) : capa determinista (extracto fuente recuperado verbatim) + capa probabilística (`langdetect`)
  - Whitelist `finish_reason` / `stop_reason` : lanzar `RuntimeError` ante cualquier estado fuera de la whitelist (truncation, content_filter, etc.)
  - `max_tokens` Claude : `4096` → `16384` (evita truncation latente en segmentos de 16k chars)
  - Segmentación heading-aware : prioridad H2/H3 en la 2.ª mitad del segmento (cada segmento comienza por una sección semántica completa)
  - Propagación de errores hasta un exit code no cero : `translate_markdown_file` devuelve un estado tipado `success` / `failure` / `skipped`, `main()` `sys.exit(1)` si al menos un archivo falló (single-file y batch)
  - Añadida la dependencia `langdetect==1.0.9`
  - Tests de regresión (`tests/test_silent_failure.py`, `unittest` stdlib) cubriendo los seis eslabones de la cadena de error
