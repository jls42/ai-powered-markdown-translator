- **1.7.3** Zestaw narzędzi jakości pre-commit (2026-04-30) :
  - Konfiguracja `pre-commit` „pełny typ EurekAI” : 14 hooków podzielonych na dwa etapy (szybki pre-commit + ciężki pre-push)
  - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (chronione 4 klucze API), Lizard (CCN ≤ 12), pre-commit-hooks v5 (białe znaki, EOF, duże pliki, shebangi itd.)
  - Pre-push : mypy (stopniowy tryb łagodny), Opengrep SAST (translate.py + scripts/), pip-audit (tryb początkowego raportowania), unittest discover (tests/ + scripts/tests/)
  - Lokalne wrappery w `scripts/`, które używają `./venv/bin/python` (system nie ma `python` „gołego” poza venv)
  - `scripts/audit_verdict.py` : parser JSON pip-audit z 11 testami unittest, port Pythona dostosowany z parsera jls42-astro
  - Naprawiono 7 początkowych naruszeń ruff : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
  - Dokumentacja : README.md (FR) + CLAUDE.md (szczegółowy workflow), 28 wygenerowanych ponownie tłumaczeń
  - Lizard tymczasowo wyklucza `translate.py` (4 funkcje o CCN 21-47, refaktoryzacja planowana w osobnym PR) — ścisła bramka dla scripts/, aby uniknąć regresji
- **1.7.2** Naprawa cichego niepowodzenia przy długich tłumaczeniach (2026-04-28) :
  - Walidacja języka po tłumaczeniu u wszystkich dostawców (OpenAI, Mistral, Claude, Gemini) : warstwa deterministyczna (odzyskany dosłownie fragment źródłowy) + warstwa probabilistyczna (`langdetect`)
  - Biała lista `finish_reason` / `stop_reason` : zgłaszanie `RuntimeError` dla każdego stanu spoza białej listy (truncation, content_filter itp.)
  - `max_tokens` Claude : `4096` → `16384` (unika ukrytego truncation na segmentach 16k znaków)
  - Segmentacja uwzględniająca nagłówki: priorytet H2/H3 w drugiej połowie segmentu (każdy segment zaczyna się od kompletnej sekcji semantycznej)
  - Propagacja błędów aż do niezerowego exit code: `translate_markdown_file` zwraca status typowany `success` / `failure` / `skipped`, `main()` `sys.exit(1)`, jeśli co najmniej jeden plik zakończył się niepowodzeniem (single-file i batch)
  - Dodano zależność `langdetect==1.0.9`
  - Testy regresji (`tests/test_silent_failure.py`, `unittest` stdlib) obejmujące sześć ogniw łańcucha błędów
