- **1.7.3** Narzędzia jakości pre-commit (2026-04-30) :
  - Setup `pre-commit` „type EurekAI complet” : 14 hooków rozłożonych na dwa etapy (szybki pre-commit + ciężki pre-push)
  - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 chronione klucze API), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, etc.)
  - Pre-push : mypy (stopniowy, łagodny tryb), Opengrep SAST (translate.py + scripts/), pip-audit (tryb początkowego raportowania), unittest discover (tests/ + scripts/tests/)
  - Lokalne wrappery w `scripts/`, które używają `./venv/bin/python` (system nie ma `python` „na czysto” poza venv)
  - `scripts/audit_verdict.py` : parser JSON pip-audit z 11 testami unittest, dostosowany port w Pythonie parsera jls42-astro
  - 7 początkowych naruszeń ruff naprawionych : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
  - Dokumentacja : README.md (FR) + CLAUDE.md (szczegółowy workflow), 28 odtworzonych tłumaczeń
  - Lizard tymczasowo wyklucza `translate.py` (4 funkcje o CCN 21-47, refaktoryzacja zaplanowana w osobnym PR) — ścisła bramka na scripts/ aby uniknąć regresji
- **1.7.2** Naprawa cichego niepowodzenia przy długich tłumaczeniach (2026-04-28) :
  - Walidacja języka po tłumaczeniu u wszystkich dostawców (OpenAI, Mistral, Claude, Gemini) : warstwa deterministyczna (odzyskany dosłownie fragment źródła) + warstwa probabilistyczna (`langdetect`)
  - Whitelist `finish_reason` / `stop_reason` : zgłaszanie `RuntimeError` dla każdego stanu spoza whitelisty (truncation, content_filter, etc.)
  - `max_tokens` Claude : `4096` → `16384` (unika utajonego truncation na segmentach 16k znaków)
  - Segmentacja uwzględniająca nagłówki : priorytet H2/H3 w drugiej połowie segmentu (każdy segment zaczyna się od kompletnej sekcji semantycznej)
  - Propagacja błędów aż do niezerowego kodu wyjścia : `translate_markdown_file` zwraca typowany status `success` / `failure` / `skipped`, `main()` `sys.exit(1)` jeśli co najmniej jeden plik zakończył się niepowodzeniem (single-file i batch)
  - Dodanie zależności `langdetect==1.0.9`
  - Testy regresji (`tests/test_silent_failure.py`, `unittest` stdlib) obejmujące sześć ogniw łańcucha błędów
