- **1.9** サイレント失敗の修正 + 完全な品質ツール群 (2026-05-03) :
  - **長文翻訳におけるサイレント失敗の修正** :
    - すべてのプロバイダー（OpenAI、Mistral、Claude、Gemini）で翻訳後の言語検証：決定論的レイヤー（ソースの抜粋が逐語的に再検出される）+ 確率的レイヤー (`langdetect`)
    - `finish_reason` / `stop_reason` のホワイトリスト：ホワイトリスト外のあらゆる状態（truncation、content_filter など）で `RuntimeError` を発生させる
    - `max_tokens` Claude：`4096` → `16384`（16k 文字セグメントでの潜在的な truncation を回避）
    - 見出しを意識したセグメンテーション：セグメントの後半で H2/H3 を優先（各セグメントは完全な意味単位のセクションから開始）
    - 非ゼロの exit code までエラーを伝播：`translate_markdown_file` は型付きステータス `success` / `failure` / `skipped` を返し、`main()` `sys.exit(1)` は少なくとも 1 ファイルが失敗した場合に発生（単一ファイルおよびバッチ）
    - すべてのプロバイダーで empty-content ガード、ソース/出力の sanity ratio（≥ 500 文字、< 5% = 拒否）、プレースホルダーコードの検証 (`#CODEBLOCK`/`#INLINECODE`)、LLM 後の正規化（見出しにくっついた区切り/リンク）、`BadRequestError` 再試行、`reasoning_effort` なし
    - 依存関係 `langdetect==1.0.9` を追加
  - **品質ツール群 pre-commit**（「完全な EurekAI 型」、14 hooks） :
    - Pre-commit : ruff（lint+format）、shellcheck、prettier（md/yaml/json）、detect-secrets（4 つの API key を保護）、Lizard（CCN ≤ 12）、pre-commit-hooks v5（whitespace、EOF、large-files、shebangs など）
    - Pre-push : mypy（段階的に緩いモード）、Opengrep SAST（translate.py + scripts/）、pip-audit（初期 reporting モード）、unittest discover（tests/ + scripts/tests/）
    - `scripts/` 内のローカルラッパーは `./venv/bin/python` を使用
    - `scripts/audit_verdict.py` : 11 の unittest テストを備えた pip-audit の JSON パーサー、jls42-astro のパーサーをベースにした Python 版
    - 初期の ruff 違反 7 件を修正：B904（raise from）×2、B007（unused dirs）、C408（dict literal）、C419（list-comp）、SIM105（contextlib.suppress）、SIM110（any()）
    - Lizard は一時的に `translate.py` を除外（CCN 21-47 の 4 関数、リファクタリング予定）— scripts/ に対して厳格なゲート
  - **SonarCloud + 網羅的カバレッジ** :
    - GitHub Actions ワークフロー `SonarCloud`（sonarcloud.yml + sonar-project.properties）: 各 push と pull-request で解析、`coverage.xml` 経由で coverage
    - README の上部に 11 個の SonarCloud バッジ（Quality Gate、Security/Reliability/Maintainability ratings、Coverage、Vulnerabilities、Bugs、Code Smells、Duplicated Lines、Technical Debt、Lines of Code）
    - `tests/test_silent_failure.py`（`unittest` stdlib）: サイレント失敗のエラーチェーンの 6 つのリンクをカバー
    - `tests/test_orchestration.py`（+79 tests）: `translate.py` のオーケストレーション層をカバー（`_resolve_*_filename`、`_existing_translation_exists`、`_record_translation_status`、`_write_output_file`、`translate_directory`、`_validate_input_paths`、`_init_*_client`、`_select_provider_client`、`_normalize_collapsed_markdown`、`_cleanup_source_flag`、`_validate_news_flags_*`、`_openai_create_with_fallback` TypeError + BadRequestError のフォールバック、o1-series のプロンプト形式、`_validate_translation_output` の early-return 分岐）
    - `scripts/tests/test_audit_verdict.py` : `main()`（stdin/stdout）と `if __name__ == "__main__"` のブロックを subprocess 経由でカバー
    - **新規コードの Coverage** : 75.5% → 約 98%（translate.py 98%、scripts/audit_verdict.py 97%）
  - ドキュメント：バッジ付き `README.md`（FR + 14 翻訳）、`CLAUDE.md`（pre-commit ワークフロー + 詳細な watch CI）、28 個の翻訳を再生成
