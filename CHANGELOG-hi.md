- **1.9** silent-failure + पूर्ण गुणवत्ता टूलिंग का सुधार (2026-05-03) :
  - **लंबे अनुवादों पर silent-failure का सुधार** :
    - अनुवाद के बाद सभी providers (OpenAI, Mistral, Claude, Gemini) पर भाषा सत्यापन : deterministic परत (स्रोत अंश verbatim मिला) + probabilistic परत (`langdetect`)
    - `finish_reason` / `stop_reason` whitelist : whitelist से बाहर की किसी भी स्थिति पर `RuntimeError` फेंकें (truncation, content_filter, आदि.)
    - `max_tokens` Claude : `4096` → `16384` (16k chars के segments पर latent truncation से बचता है)
    - heading-aware segmentation : segment के दूसरे आधे भाग में H2/H3 को प्राथमिकता (प्रत्येक segment एक पूर्ण semantic section से शुरू होता है)
    - त्रुटियों का propagation non-zero exit code तक : `translate_markdown_file` एक typed status `success` / `failure` / `skipped` लौटाता है, `main()` `sys.exit(1)` यदि कम से कम एक फ़ाइल विफल हुई हो (single-file और batch)
    - सभी providers पर empty-content guard, source/output sanity ratio (≥ 500 chars, < 5% = अस्वीकार), placeholders code validation (`#CODEBLOCK`/`#INLINECODE`), post-LLM normalization (heading से चिपके separators/links), `BadRequestError` retry बिना `reasoning_effort`
    - निर्भरता `langdetect==1.0.9` जोड़ी गई
  - **pre-commit गुणवत्ता टूलिंग** ("type EurekAI complet", 14 hooks) :
    - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 API keys सुरक्षित), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, आदि.)
    - Pre-push : mypy (क्रमिक lax mode), Opengrep SAST (translate.py + scripts/), pip-audit (प्रारंभिक reporting mode), unittest discover (tests/ + scripts/tests/)
    - `scripts/` में स्थानीय wrappers जो `./venv/bin/python` का उपयोग करते हैं
    - `scripts/audit_verdict.py` : 11 unittest tests के साथ pip-audit JSON parser, jls42-astro parser से अनुकूलित Python port
    - 7 प्रारंभिक ruff violations सुधारे गए : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
    - Lizard अस्थायी रूप से `translate.py` को बाहर रखता है (CCN 21-47 वाली 4 functions, refactor planned) — scripts/ पर strict gate
  - **SonarCloud + पूर्ण कवरेज** :
    - GitHub Actions workflow `SonarCloud` (sonarcloud.yml + sonar-project.properties) : हर push और pull-request पर analysis, coverage via `coverage.xml`
    - README के शीर्ष पर SonarCloud के 11 badges (Quality Gate, Security/Reliability/Maintainability ratings, Coverage, Vulnerabilities, Bugs, Code Smells, Duplicated Lines, Technical Debt, Lines of Code)
    - `tests/test_silent_failure.py` (`unittest` stdlib) : silent-failure error chain के छह links को कवर करता है
    - `tests/test_orchestration.py` (+79 tests) : `translate.py` की orchestration layer को कवर करता है (`_resolve_*_filename`, `_existing_translation_exists`, `_record_translation_status`, `_write_output_file`, `translate_directory`, `_validate_input_paths`, `_init_*_client`, `_select_provider_client`, `_normalize_collapsed_markdown`, `_cleanup_source_flag`, `_validate_news_flags_*`, `_openai_create_with_fallback` TypeError + BadRequestError fallbacks, o1-series prompt format, `_validate_translation_output` की early-return branches)
    - `scripts/tests/test_audit_verdict.py` : `main()` (stdin/stdout) और subprocess के माध्यम से `if __name__ == "__main__"` ब्लॉक का कवरेज
    - **नए कोड पर कवरेज** : 75.5% → ~98% (translate.py 98%, scripts/audit_verdict.py 97%)
  - दस्तावेज़ीकरण : badges के साथ `README.md` (FR + 14 translations), `CLAUDE.md` (pre-commit workflow + detailed watch CI), 28 regenerated translations
