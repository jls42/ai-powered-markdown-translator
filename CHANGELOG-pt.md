- **1.9** Corrige falha silenciosa + kit de qualidade completo + nota de tradução multi-posicionamento (2026-05-07) :
  - **Nota de tradução multi-posicionamento + marcador de formato "embed card"** :
    - Novas opções de CLI (aditivas, padrões inalterados → **não breaking**) :
      - `--note_position {top,bottom,both}` (padrão : `bottom`) : coloca a nota no topo, na base, ou em ambos os locais do arquivo traduzido.
      - `--note_format {legacy,marker}` (padrão : `legacy`) :
        - `legacy` reproduz estritamente o comportamento v1.8 (parágrafo em negrito `**…**`) **byte-for-byte**.
        - `marker` emite uma definição de referência de link Markdown invisível (`[ai-translation-note-<placement>]: <> "v=1 source=… target=… model=… date=…"`) seguida de um **blockquote de 3 parágrafos** estruturado para uma renderização tipo "GitHub repo embed card" : título do projeto em código inline (`**\`ai-powered-markdown-translator\`\*\*`), descrição traduzida pelo LLM, e link CTA (`[Voir le projet sur GitHub ↗](URL)`) com seta visível. Utilizável no build por um plugin remark (cf. blog jls42.org → plugin `remark-translation-banner`).
    - **Invariantes nunca enviados ao LLM** : o título do repo e a URL GitHub são montados do lado Python após a tradução da frase descritiva. O LLM nunca vê o slug `ai-powered-markdown-translator` nem `https://github.com/jls42/...`, garantindo que nenhum renderer/case/scheme seja alterado.
    - **Inserção consciente do frontmatter** : no modo `top` ou `both`, a nota é inserida **após o bloco `---` de fechamento** do frontmatter YAML (segurança Astro Content Collections / gray-matter). O helper `_split_frontmatter` detecta `---\n…\n---\n` no início do arquivo e preserva sua integridade ; **levanta `RuntimeError`** em frontmatter aberto sem fence de fechamento (o arquivo sobe em `failed_files` em vez de ser escrito com uma nota mal posicionada).
    - **Sanitizador de modelo com lista de permissões** : `_sanitize_model` substitui qualquer caractere fora de `[A-Za-z0-9._:/-]` por `_`, fallback `unknown` se vazio. Alinha-se ao validador do lado do plugin remark Astro e neutraliza os caracteres que quebrariam o formato do marcador (espaço, aspas, parêntese, vírgula, etc.).
    - **Refatoração interna** : `_append_translation_note` (1 função monolítica) → 7 helpers puros (`_translation_note_invariants`, `_build_translation_note_phrase`, `_assemble_translation_note_paragraphs`, `_build_translation_note_source`, `_sanitize_model`, `_quote_lines`, `_split_frontmatter`, `_build_translation_note_block`, `_compose_with_notes`). Builder/composer separados (o builder retorna um bloco puro sem separador, o composer aplica os `\n\n` conforme a posição) ; produção e helper source compartilham o mesmo montador de 3 parágrafos.
    - **`_quote_lines` preservando linhas em branco** : prefixa cada linha com `> `, transformando as linhas vazias em `>` sozinho. Permite ao mdast ver 3 parágrafos distintos no blockquote (título / descrição / link) em vez de um único parágrafo com quebras de linha.
    - **`_build_translation_note_block` adaptativo** : de acordo com o número de parágrafos que o LLM preservou (3 = formato de card completo, 2 = frase + link, 1 = fallback). O fallback de 1 parágrafo **não envolve mais em `**...**`** quando um link Markdown `](` é detectado (renderização frágil de `<strong>` em torno de um link).
    - **Compatibilidade retroativa** : `getattr(args, "note_position", "bottom")` e `getattr(args, "note_format", "legacy")` do lado de `_compose_with_notes` — os Namespace sem esses atributos (testes existentes, chamadas programáticas externas) continuam funcionando sem modificação.
  - **Corrige falha silenciosa em traduções longas** :
    - Validação de idioma pós-tradução em todos os provedores (OpenAI, Mistral, Claude, Gemini) : camada determinística (trecho de origem recuperado verbatim) + camada probabilística (`langdetect`)
    - Lista de permissões `finish_reason` / `stop_reason` : levantar `RuntimeError` em qualquer estado fora da lista de permissões (truncation, content_filter, etc.)
    - `max_tokens` Claude : `4096` → `32768` (evita truncation latente em segmentos 16k, margem cross-script FR→JA/ZH/KO/AR/HI)
    - Segmentação ciente de headings : prioridade H2/H3 na 2ª metade do segmento (cada segmento começa por uma seção semântica completa)
    - Propagação dos erros até código de saída não zero : `translate_markdown_file` retorna um status tipado `success` / `failure` / `skipped`, `main()` `sys.exit(1)` se ao menos um arquivo falhar (single-file e batch)
    - Guard de conteúdo vazio em todos os provedores, razão de sanidade source/output (≥ 500 chars, < 5% = recusa), validação de placeholders code (`#CODEBLOCK`/`#INLINECODE`), normalização pós-LLM (separadores/links colados a um heading), `BadRequestError` retry sem `reasoning_effort`
    - Adição da dependência `langdetect==1.0.9`
