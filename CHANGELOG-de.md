- **1.7.4** SonarCloud + vollständige Abdeckung (2026-05-03) :
  - GitHub-Actions-Workflow `SonarCloud` (sonarcloud.yml + sonar-project.properties) : Analyse bei jedem Push und jeder Pull-Request, Coverage-Berechnung über `coverage.xml`
  - 11 SonarCloud-Badges oben im README (Quality Gate, Security/Reliability/Maintainability ratings, Coverage, Vulnerabilities, Bugs, Code Smells, Duplicated Lines, Technical Debt, Lines of Code)
  - Neue Datei `tests/test_orchestration.py` (+79 Tests), die die Orchestrierungsschicht von `translate.py` abdeckt : `_resolve_*_filename`, `_existing_translation_exists`, `_record_translation_status`, `_write_output_file`, `translate_directory`, `_validate_input_paths`, `_init_*_client`, `_select_provider_client`, `_normalize_collapsed_markdown`, `_cleanup_source_flag`, `_validate_news_flags_*`, `_openai_create_with_fallback` (TypeError + BadRequestError-Fallbacks), o1-series-Prompt-Format, Early-Return-Zweige von `_validate_translation_output`
  - `scripts/tests/test_audit_verdict.py` erweitert : Abdeckung von `main()` (stdin/stdout) und des `if __name__ == "__main__"`-Blocks über subprocess
  - **Coverage für neuen Code** : 75.5% → ~98% (translate.py 98%, scripts/audit_verdict.py 97%)
  - Zusätzliche Härtung von `translate.py` (PR-Review-Feedback) : Empty-Content-Guard bei allen Providern, Plausibilitätsprüfung des Quell-/Ausgabe-Verhältnisses (≥ 500 Zeichen, < 5% = Ablehnung), Validierung von Platzhalter-Code (`#CODEBLOCK`/`#INLINECODE`), Normalisierung nach LLM (getrennte/verbundene Links an einer Überschrift), `BadRequestError`-Retry ohne `reasoning_effort`
- **1.7.3** Pre-commit-Qualitäts-Tooling (2026-04-30) :
  - Setup `pre-commit` „vollständiger EurekAI-Typ“ : 14 Hooks auf zwei Stufen verteilt (schnelles pre-commit + schweres pre-push)
  - Pre-commit : ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 geschützte API-Keys), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs usw.)
  - Pre-push : mypy (schrittweiser Lax-Modus), Opengrep SAST (translate.py + scripts/), pip-audit (initialer Reporting-Modus), unittest discover (tests/ + scripts/tests/)
  - Lokale Wrapper in `scripts/`, die `./venv/bin/python` verwenden (das System hat nicht `python` nackt außerhalb von venv)
  - `scripts/audit_verdict.py` : JSON-Parser für pip-audit mit 11 unittest-Tests, angepasster Python-Port des jls42-astro-Parsers
  - 7 anfängliche ruff-Verstöße behoben : B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
  - Dokumentation : README.md (FR) + CLAUDE.md (detaillierter Workflow), 28 neu generierte Übersetzungen
  - Lizard schließt vorübergehend `translate.py` aus (4 Funktionen mit CCN 21-47, Refactoring in einem eigenen PR geplant) — striktes Gate für scripts/, um Regressionen zu vermeiden
- **1.7.2** Behebung von Silent-Failure bei langen Übersetzungen (2026-04-28) :
  - Sprachvalidierung nach der Übersetzung bei allen Providern (OpenAI, Mistral, Claude, Gemini) : deterministische Ebene (Quelltextauszug wörtlich wiedergefunden) + probabilistische Ebene (`langdetect`)
  - Whitelist `finish_reason` / `stop_reason` : `RuntimeError` bei jedem Zustand außerhalb der Whitelist auslösen (truncation, content_filter usw.)
  - `max_tokens` Claude : `4096` → `16384` (vermeidet latente Trunkierung bei 16k-Zeichen-Segmenten)
  - Heading-aware-Segmentierung : Priorität H2/H3 in der zweiten Hälfte des Segments (jedes Segment beginnt mit einem vollständigen semantischen Abschnitt)
  - Fehlerweitergabe bis zum Nicht-Null-Exit-Code : `translate_markdown_file` gibt einen typisierten Status `success` / `failure` / `skipped` zurück, `main()` `sys.exit(1)` wenn mindestens eine Datei fehlgeschlagen ist (Einzeldatensatz und Batch)
  - Abhängigkeit `langdetect==1.0.9` hinzugefügt
  - Regressionstests (`tests/test_silent_failure.py`, `unittest` stdlib), die die sechs Glieder der Fehlerkette abdecken
