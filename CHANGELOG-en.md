- **1.9** Silent-failure fix + full quality tooling + multi-position translation note (2026-05-07):
  - **Multi-position translation note + "embed card" marker format**:
    - New CLI options (additive, defaults unchanged → **non breaking**):
      - `--note_position {top,bottom,both}` (default: `bottom`): places the note at the top, bottom, or both places in the translated file.
      - `--note_format {legacy,marker}` (default: `legacy`):
        - `legacy` strictly reproduces v1.8 behavior (bold paragraph `**…**`) **byte-for-byte**.
        - `marker` emits an invisible Markdown link reference definition (`[ai-translation-note-<placement>]: <> "v=1 source=… target=… model=… date=…"`) followed by a structured **3-paragraph blockquote** for a "GitHub repo embed card" style render: project title in inline code (`**\`ai-powered-markdown-translator\`\*\*`), description translated by the LLM, and CTA link (`[Voir le projet sur GitHub ↗](URL)`) with visible arrow. Usable at build time by a remark plugin (see jls42.org blog → `remark-translation-banner` plugin).
    - **Invariants never sent to the LLM**: repo title and GitHub URL are assembled on the Python side after translating the descriptive sentence. The LLM never sees slug `ai-powered-markdown-translator` or `https://github.com/jls42/...`, ensuring that no renderer/case/scheme is altered.
    - **Frontmatter-aware insertion**: in `top` or `both` mode, the note is inserted **after the closing `---` block** of the YAML frontmatter (Astro Content Collections / gray-matter safety). Helper `_split_frontmatter` detects `---\n…\n---\n` at the start of the file and preserves its integrity; **raises `RuntimeError`** on an open frontmatter without a closing fence (the file is sent back to `failed_files` instead of being written with a misplaced note).
    - **Whitelist model sanitizer**: `_sanitize_model` replaces every character outside `[A-Za-z0-9._:/-]` with `_`, fallback `unknown` if empty. Aligned with the validator on the Astro remark plugin side and neutralizes characters that would break the marker format (space, quote, parenthesis, comma, etc.).
    - **Internal refactor**: `_append_translation_note` (1 monolithic function) → 7 pure helpers (`_translation_note_invariants`, `_build_translation_note_phrase`, `_assemble_translation_note_paragraphs`, `_build_translation_note_source`, `_sanitize_model`, `_quote_lines`, `_split_frontmatter`, `_build_translation_note_block`, `_compose_with_notes`). Builder/composer separated (the builder returns a pure block without separator, the composer applies the `\n\n` according to position); production and source helper share the same 3-paragraph assembler.
    - **`_quote_lines` blank-preserving**: prefixes each line with `> `, turning blank lines into `>` only. Allows mdast to see 3 distinct paragraphs in the blockquote (title / description / link) instead of a single paragraph with line breaks.
    - **Adaptive `_build_translation_note_block`**: depending on the number of paragraphs preserved by the LLM (3 = complete card format, 2 = sentence + link, 1 = fallback). The 1-paragraph fallback **no longer wraps in `**...**`** when a Markdown link `](` is detected (fragile rendering of `<strong>` around a link).
    - **Backward compatibility**: `getattr(args, "note_position", "bottom")` and `getattr(args, "note_format", "legacy")` on the `_compose_with_notes` side — Namespaces without these attributes (existing tests, external programmatic calls) continue to work unchanged.
  - **Silent-failure fix on long translations**:
    - Post-translation language validation on all providers (OpenAI, Mistral, Claude, Gemini): deterministic layer (source excerpt recovered verbatim) + probabilistic layer (`langdetect`)
    - `finish_reason` / `stop_reason` whitelist: raise `RuntimeError` on any state outside the whitelist (truncation, content_filter, etc.)
    - `max_tokens` Claude: `4096` → `32768` (avoids latent truncation on 16k segments, cross-script margin FR→JA/ZH/KO/AR/HI)
    - Heading-aware segmentation: priority H2/H3 in the second half of the segment (each segment starts with a complete semantic section)
    - Error propagation up to a non-zero exit code: `translate_markdown_file` returns a typed status `success` / `failure` / `skipped`, `main()` `sys.exit(1)` if at least one file failed (single-file and batch)
    - Empty-content guard on all providers, source/output sanity ratio (≥ 500 chars, < 5% = refusal), code placeholder validation (`#CODEBLOCK`/`#INLINECODE`), post-LLM normalization (separators/links glued to a heading), `BadRequestError` retry without `reasoning_effort`
    - Added dependency `langdetect==1.0.9`
  - **Pre-commit quality tooling** ("full EurekAI type", 14 hooks):
    - Pre-commit: ruff (lint+format), shellcheck, prettier (md/yaml/json), detect-secrets (4 protected API keys), Lizard (CCN ≤ 12), pre-commit-hooks v5 (whitespace, EOF, large-files, shebangs, etc.)
    - Pre-push: mypy (progressive lax mode), Opengrep SAST (translate.py + scripts/), pip-audit (initial reporting mode), unittest discover (tests/ + scripts/tests/)
    - Local wrappers in `scripts/` that use `./venv/bin/python`

    - 7 initial ruff violations fixed: B904 (raise from) ×2, B007 (unused dirs), C408 (dict literal), C419 (list-comp), SIM105 (contextlib.suppress), SIM110 (any())
