- **1.9** silent-failure 수정 + 완전한 품질 도구화 + 다중 위치 번역 노트 (2026-05-07) :
  - **다중 위치 번역 노트 + "embed card" 마커 형식** :
    - 새로운 CLI 옵션(추가형, 기본값 변경 없음 → **비파괴적**) :
      - `--note_position {top,bottom,both}` (기본값: `bottom`) : 번역된 파일의 상단, 하단, 또는 양쪽 모두에 노트를 배치합니다.
      - `--note_format {legacy,marker}` (기본값: `legacy`) :
        - `legacy`는 v1.8의 동작을 **바이트 단위로 동일하게** 그대로 재현합니다(`**…**`).
        - `marker`는 보이지 않는 Markdown 링크 참조 정의(`[ai-translation-note-<placement>]: <> "v=1 source=… target=… model=… date=…"`)를 출력한 뒤, **3문단 blockquote**로 구성된 "GitHub repo embed card" 스타일 렌더링을 생성합니다: 프로젝트 제목을 인라인 코드(`**\`ai-powered-markdown-translator\`\*\*`)로 표시하고, LLM이 번역한 설명과, 화살표가 보이는 CTA 링크(`[Voir le projet sur GitHub ↗](URL)`)를 포함합니다. 빌드 시 remark 플러그인으로 활용할 수 있습니다(참조: blog jls42.org → 플러그인 `remark-translation-banner`).
    - **LLM에 절대 전송되지 않는 불변값** : repo 제목과 GitHub URL은 설명 문장을 번역한 뒤 Python 쪽에서 조립합니다. LLM은 슬러그 `ai-powered-markdown-translator`나 `https://github.com/jls42/...`를 절대 보지 않으므로, 렌더러/대소문자/스킴이 변경될 일이 없습니다.
    - **Frontmatter-aware 삽입** : `top` 또는 `both` 모드에서는 노트가 YAML frontmatter의 닫는 `---` 블록 **뒤에** 삽입됩니다(Astro Content Collections / gray-matter 안전성). 헬퍼 `_split_frontmatter`는 파일 시작의 `---\n…\n---\n`를 감지하고 무결성을 보존하며, 닫는 fence가 없는 열린 frontmatter에서는 **`RuntimeError`를 발생**시킵니다(이 경우 파일은 잘못 배치된 노트와 함께 기록되는 대신 `failed_files`로 다시 올라갑니다).
    - **화이트리스트 모델 Sanitizer** : `_sanitize_model`는 `[A-Za-z0-9._:/-]` 바깥의 모든 문자를 `_`로 치환하고, 비어 있으면 `unknown`로 대체합니다. Astro remark 플러그인 측 검증기와 맞추고, 마커 형식을 깨뜨리는 문자(공백, 따옴표, 괄호, 쉼표 등)를 무력화합니다.
    - **내부 리팩터링** : `_append_translation_note`(1개의 모놀리식 함수) → 7개의 순수 헬퍼(`_translation_note_invariants`, `_build_translation_note_phrase`, `_assemble_translation_note_paragraphs`, `_build_translation_note_source`, `_sanitize_model`, `_quote_lines`, `_split_frontmatter`, `_build_translation_note_block`, `_compose_with_notes`). builder/composer를 분리했고(builder는 구분자 없는 순수 블록을 반환, composer는 위치에 따라 `\n\n`를 적용), production과 source helper는 동일한 3문단 어셈블러를 공유합니다.
    - **`_quote_lines` 공백 보존** : 각 줄 앞에 `> `를 붙이고, 빈 줄은 `>`만 남기도록 변환합니다. 이를 통해 mdast가 blockquote 안에서 제목 / 설명 / 링크의 3개 문단을 별개로 인식하게 하며, 줄바꿈이 섞인 하나의 문단으로 보이지 않게 합니다.
    - **`_build_translation_note_block` 적응형** : LLM이 보존한 문단 수에 따라 동작합니다(3 = 완전한 카드 형식, 2 = 문장 + 링크, 1 = 폴백). 1문단 폴백은 Markdown 링크 `](`가 감지되면 더 이상 `**...**`로 감싸지 않습니다(`<strong>`가 링크 주변에서 취약하게 렌더링되던 문제 방지).
    - **하위 호환성** : `getattr(args, "note_position", "bottom")`와 `getattr(args, "note_format", "legacy")`는 `_compose_with_notes` 측에서 유지됩니다. 이러한 속성이 없는 Namespace(기존 테스트, 외부 프로그램 호출)는 수정 없이 계속 동작합니다.
  - **긴 번역에서 발생하는 silent-failure 수정** :
    - 모든 provider(OpenAI, Mistral, Claude, Gemini)에 대해 번역 후 언어 검증을 수행합니다: 결정론적 레이어(원문 일부가 그대로 복원되었는지 확인) + 확률론적 레이어(`langdetect`)
    - `finish_reason` / `stop_reason` 화이트리스트: 화이트리스트 밖의 상태(truncation, content_filter 등)가 나오면 `RuntimeError`를 발생시킵니다.
    - `max_tokens` Claude : `4096` → `32768` (16k 세그먼트에서의 잠재적 truncation을 방지하고, FR→JA/ZH/KO/AR/HI 같은 교차 스크립트에 여유를 둠)
    - 헤딩 인식 세그먼테이션: 세그먼트 후반부에서 H2/H3를 우선 처리합니다(각 세그먼트가 완전한 의미 단위로 시작되도록).
    - 오류를 non-zero exit code까지 전파: `translate_markdown_file`는 타입이 지정된 상태 `success` / `failure` / `skipped`를 반환하고, 하나 이상의 파일이 실패하면 `main()` `sys.exit(1)`가 적용됩니다(single-file 및 batch 모두).
    - 모든 provider에 대한 empty-content 가드, 소스/출력 정합성 비율(≥ 500자, < 5% = 거부), 코드 placeholder 검증(`#CODEBLOCK`/`#INLINECODE`), LLM 후 정규화(구분자/링크가 헤딩에 붙는 현상 수정), `BadRequestError` 및 `reasoning_effort` 없이 retry
    - 의존성 `langdetect==1.0.9` 추가
  - **사전 커밋 품질 도구화** ("type EurekAI complet", 14 hooks) :

    - `scripts/audit_verdict.py` : 11개의 unittest 테스트가 있는 pip-audit JSON 파서, jls42-astro 파서의 Python 포팅 버전
