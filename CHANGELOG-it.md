- **1.9.2** Correzione dell'estrazione dell'URL di attribuzione delle news con parentesi annidate o prefisso FR (2026-05-11) :

  - **Bug corretto** : l'estrazione degli URL di attribuzione in `_protect_news_quotes` utilizzava la regex `re.search(r"\((.+?)\)", attribution)` (cattura lazy tra parentesi). Su un'attribuzione del tipo `(relayé par [@user sur X](https://x.com/.../123))` (parentesi annidate : `(` esterna + `]()` del markdown link), la cattura si fermava alla prima `)` incontrata → stringa troncata + con il prefisso FR : `relayé par [@user sur X](https://x.com/.../123` (senza `)` finale). Conseguenza : `_validate_news_post` cercava questa stringa nell'output tradotto e falliva sistematicamente (due motivi : `)` troncato + "relayé par" tradotto in `relayed by`/`weitergeleitet von`/...). La cascata completa low → medium → high → gpt-5.5 non poteva passare.
  - **Correzione** : la regex diventa `re.search(r"\]\(([^)]+)\)", attribution)` — punta specificamente al `](url)` del markdown link, cattura **solo l'URL puro** (senza prefisso FR né troncamento), invariabile preservato dai placeholder `#URL{N}#` durante la traduzione. Robusta rispetto ai due pattern problematici :
    - `(relayé par [@account sur X](url))` — parentesi annidate
    - `via [@source](url)` o `selon [@author](url)` — prefisso FR senza parentesi esterne
  - **Test** : 2 nuovi in `test_silent_failure.py` classe `TestNewsCitationExtraction` :
    - `test_extract_attribution_url_with_nested_parens` (caso esatto riprodotto del bug Genspark CEO E2B)
    - `test_extract_attribution_url_with_french_prefix` (variante con `via`)
  - **Lacuna di copertura** : `check-editorial-coverage.py` valida la sintassi editoriale ma non la traducibilità da parte del traduttore. Un miglioramento possibile (fuori scope v1.9.2) sarebbe un check che simula l'estrazione dell'attribuzione in dry-run per rilevare i pattern a rischio PRIMA della pubblicazione.

- **1.9** Correzione silent-failure + toolchain di qualità completa + nota di traduzione multi-posizione (2026-05-07) :
  - **Nota di traduzione multi-posizione + formato marker "embed card"** :
    - Nuove opzioni CLI (additive, default invariati → **non breaking**) :
      - `--note_position {top,bottom,both}` (default : `bottom`) : posiziona la nota in alto, in basso, o in entrambi i punti del file tradotto.
      - `--note_format {legacy,marker}` (default : `legacy`) :
        - `legacy` riproduce fedelmente il comportamento v1.8 (paragrafo in grassetto `**…**`) **byte-for-byte**.
        - `marker` emette una definizione Markdown invisibile di link reference (`[ai-translation-note-<placement>]: <> "v=1 source=… target=… model=… date=…"`) seguita da un **blockquote di 3 paragrafi** strutturato per un rendering tipo "GitHub repo embed card" : titolo del progetto in codice inline (`**\`ai-powered-markdown-translator\`\*\*`), descrizione tradotta dal LLM, e link CTA (`[Voir le projet sur GitHub ↗](URL)`) con freccia visibile. Utilizzabile in fase di build da un plugin remark (cf. blog jls42.org → plugin `remark-translation-banner`).
    - **Invarianti mai inviati al LLM** : titolo del repo e URL GitHub sono assemblati lato Python dopo la traduzione della frase descrittiva. Il LLM non vede mai né lo slug `ai-powered-markdown-translator` né `https://github.com/jls42/...`, garantendo che nessun renderer/case/scheme venga alterato.
    - **Inserimento consapevole del frontmatter** : in modalità `top` o `both`, la nota viene inserita **dopo il blocco `---` di chiusura** del frontmatter YAML (sicurezza Astro Content Collections / gray-matter). Helper `_split_frontmatter` rileva `---\n…\n---\n` all'inizio del file e ne preserva l'integrità ; **solleva `RuntimeError`** su frontmatter aperto senza fence di chiusura (il file risale in `failed_files` invece di essere scritto con una nota mal posizionata).
    - **Sanitizer modello whitelist** : `_sanitize_model` sostituisce ogni carattere fuori da `[A-Za-z0-9._:/-]` con `_`, fallback `unknown` se vuoto. Si allinea al validatore lato plugin remark Astro e neutralizza i caratteri che romperebbero il formato del marker (spazio, virgolette, parentesi, virgola, ecc.).
    - **Refactor interno** : `_append_translation_note` (1 funzione monolitica) → 7 helper puri (`_translation_note_invariants`, `_build_translation_note_phrase`, `_assemble_translation_note_paragraphs`, `_build_translation_note_source`, `_sanitize_model`, `_quote_lines`, `_split_frontmatter`, `_build_translation_note_block`, `_compose_with_notes`). Builder/composer separati (il builder restituisce un blocco puro senza separatore, il composer applica i `\n\n` secondo la posizione) ; produzione e helper source condividono lo stesso assemblatore a 3 paragrafi.
    - **`_quote_lines` preserva gli spazi vuoti** : antepone a ogni riga `> `, trasformando le righe vuote nel solo `>`. Permette a mdast di vedere 3 paragrafi distinti nel blockquote (titolo / descrizione / link) invece di un solo paragrafo con line-break.
    - **`_build_translation_note_block` adattivo** : in base al numero di paragrafi che il LLM ha preservato (3 = formato card completo, 2 = frase + link, 1 = fallback). Il fallback a 1 paragrafo **non avvolge più in `**...**`** quando viene rilevato un link Markdown `](` (rendering fragile di `<strong>` attorno a un link).
    - **Compatibilità retroattiva** : `getattr(args, "note_position", "bottom")` e `getattr(args, "note_format", "legacy")` lato `_compose_with_notes` — i Namespace senza questi attributi (test esistenti, chiamate programmatiche esterne) continuano a funzionare senza modifiche.
  - **Correzione silent-failure sulle traduzioni lunghe** :
    - Validazione lingua post-traduzione su tutti i provider (OpenAI, Mistral, Claude, Gemini) : livello deterministico (estratto sorgente ritrovato verbatim) + livello probabilistico (`langdetect`)
    - Whitelist `finish_reason` / `stop_reason` : sollevare `RuntimeError` su qualunque stato fuori whitelist (troncamento, content_filter, ecc.)
    - `max_tokens` Claude : `4096` → `32768` (evita troncamento latente sui segmenti 16k, margine cross-script FR→JA/ZH/KO/AR/HI)
